Budget Tracker Pro - Documentation

Thank you for purchasing Budget Tracker Pro. This documentation will guide you through setting up your own Serverless Expense Manager in less than 5 minutes.

Requirement: You do NOT need a server (PHP/Node.js). You only need a free Google account for Firebase.

Features

  • ✅ Serverless: Runs entirely on Google Firebase (Free Tier).
  • ✅ PWA Ready: Installable on Android, iOS, and Desktop.
  • ✅ Cloud Sync: Real-time data sync across devices.
  • ✅ Smart Debt Manager: Track who owes you money.
  • ✅ Indian Finance: Native support for ₹ (INR), Lakhs, and Crores.

Step 1: Firebase Setup

This app uses Google Firebase for the database and authentication. It is free for small to medium apps.

1 Create Project

Go to the Firebase Console and click "Add Project". Give it any name (e.g., "MyBudget").

2 Enable Authentication

On the left sidebar, click Build -> Authentication.

  • Click "Get Started".
  • Select Google as a Sign-in provider.
  • Enable it and save.
3 Enable Database

On the left sidebar, click Build -> Firestore Database.

  • Click "Create Database".
  • Select "Start in Production Mode".
  • Choose a location nearest to you (e.g., asia-south1 for India).
4 Set Database Rules

Go to the Rules tab in Firestore and paste this security rule to ensure users only see their own data:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Step 2: Configuration

Now connect your code to your new Firebase project.

  1. In Firebase Console, click the Gear Icon ⚙️ -> Project Settings.
  2. Scroll down to "Your Apps" and click the Web () icon.
  3. Register the app (give it a nickname).
  4. Copy the firebaseConfig object shown on screen.
  5. Open config.js in your source folder and paste your keys there.
// Example config.js
const firebaseConfig = {
    apiKey: "AIzaSyD...",
    authDomain: "mybudget.firebaseapp.com",
    projectId: "mybudget-123",
    storageBucket: "mybudget.appspot.com",
    messagingSenderId: "12345...",
    appId: "1:12345..."
};

Step 3: Installation / Upload

Since this is a static HTML5 app, you can host it anywhere for free.

  • GitHub Pages: (Recommended for Developers)
  • Netlify / Vercel: Drag and drop the folder.
  • cPanel / Shared Hosting: Upload files to `public_html`.

Step 4: Domain Whitelist (Critical) ⚠️

If you skip this step, Google Login will fail with an error: auth/unauthorized-domain.

Google blocks sign-ins from unknown websites to prevent phishing. You must tell Firebase your website URL.

  1. Go to Firebase Console -> Authentication -> Settings.
  2. Click on Authorized Domains.
  3. Click "Add Domain".
  4. Enter your website domain (e.g., syedsubhani.in or my-app.netlify.app).
  5. Click Add.

Note: `localhost` is allowed by default for testing.

Step 5: PWA Setup (Optional)

To make the "Install App" prompt appear with your own logo:

  1. Replace the icons in the icons/ folder with your own (must be 192x192 and 512x512).
  2. Open manifest.json and update the "name" and "short_name" fields.

FAQ

Q: Can I change the currency to USD?

A: Yes. Open app.js and search for formatMoney. Change en-IN to en-US and ₹ to $.

Q: Where is the admin panel?

A: There is no admin panel. You can view/manage all user data directly inside the Firebase Console (Firestore Database).

Q: Is the data secure?

A: Yes. All data is stored on Google's servers. Authentication is handled via Google OAuth.
