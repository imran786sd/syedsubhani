Budget Tracker Pro - Documentation

Thank you for purchasing Budget Tracker Pro! This guide will help you set up your own Personal Finance PWA with Google Cloud Sync.

Requirements: You do not need a backend server (Node/PHP). You only need a standard web host (Netlify, Vercel, cPanel) and a free Google Firebase account.

1. Project Structure

Once you unzip the download package, you will see the following files:

  • index.html - The main structure of the app.
  • style.css - All styling, themes (Dark/Light), and responsive design.
  • app.js - The core logic (Charts, Calculations, UI Rendering).
  • firebase-init.js - Handles the connection to Google services.
  • config.js - IMPORTANT: This is where you put your API Keys.

2. Getting Started (Firebase Setup)

This app uses Google Firebase for authentication and database storage. It is free for small to medium apps.

Step A: Create Project

  1. Go to Firebase Console and click Add Project.
  2. Give it a name (e.g., "MyBudgetApp") and disable Google Analytics (optional).
  3. Click Create Project.

Step B: Enable Authentication

  1. In the Firebase sidebar, click Build > Authentication.
  2. Click Get Started.
  3. Select Google from the Sign-in providers list.
  4. Click Enable, select your support email, and click Save.

Step C: Create Database

  1. In the sidebar, click Build > Firestore Database.
  2. Click Create Database.
  3. Choose a location close to you (e.g., nam5 or asia-south1).
  4. Crucial: Start in Production Mode.

Step D: Copy API Keys

  1. Click the Gear Icon (Settings) next to "Project Overview".
  2. Scroll down to "Your apps" and click the </> (Web) icon.
  3. Register the app (e.g., "Budget Web").
  4. You will see a code block with const firebaseConfig = { ... }.
  5. Copy ONLY the keys inside the brackets.
  6. Open your config.js file and paste them there.

3. Security Rules (Mandatory)

To ensure user data is private, you must set up database rules.

  1. Go to Firestore Database > Rules tab.
  2. Delete everything there and paste the code below:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Allow users to read/write ONLY their own data
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Click Publish.
